ti is a simple (very slightly) command line time tracker. It is completely written in bash and has (almost) complete test coverage. It is inspired by timed, which I felt a bit too limiting in features. It also takes inspirateion from the simplicity of t.

If a time tracker tool makes me think for more than 10 seconds, I usually forget what I was concentrating on. This is why I created ti. With ti, you'll be only as fast as you can type, which you should be good with anyway.

The most important part about ti is that it provides just a few commands to manage your time tracking and gets out of your way. Better yet, its tiny core is extendable so you can write your own actions to make you even faster for stuff that you frequently do. In that spirit, all data is saved in an easily readable format by all humans, pythons, rubies and other pearls.

Oh and by the way, the source is just a ~150 lines bash script, so if you know bash, I suggest you scim over it to get a better feel of how this works.

Usage

Here's the minimal usage style:

$ ti on my-project
Start working on my-project.

$ ti status
You have been working on my-project for less than a minute.

$ ti fin
So you stopped working on my-project.

on and fin can take a time at which to apply the action

$ ti on another-project 2 hours ago
Start working on another-project.

$ ti st
You have been working on another-project for about 2 hours.

$ ti fin 30 minutes ago
So you stopped working on another-project.

Also illustrating in the previous example is any unambiguous prefix (st) of available actions can be used instead of the whole action name.

Put brief notes on what you've been doing.

$ ti note waiting for Napoleon to take over the world
$ ti n another simple note for demo purposes

Tag your activities for fun and profit.

$ ti tag imp

Do plugins. For those that need the extra sauce.

$ cat <<EOF > ~/.ti/greeter.ti.sh
> action-greet () {
>     echo Hello there awesome ti user!
> }
> EOF

$ ti greet # or just `ti gr`
Hello there awesome ti user!

That might not look like much, but with this simple exendability and using the API (docs coming soon, for now you'll have to check the source), you can create innovative actions. Stuff that analyse the time sheet and give a report, make a report of a week and mail that to yourself or your boss, simply echo the current project name so you can use it to show info in your terminal prompt, what not?

Status

The project is still in its infancy. I'd like for people to check it out and play with it. Suggest more awesomeness and/or contribute otherwise.

Gimme!

You can get ti from the source on bitbucket. Put the `ti` bash script (located in the bin directory) somewhere in your $PATH and make sure it has executable permissions and you should be able to use it just fine.

For the fancy relative time messages shown in the above usage examples, you'll need the dtime executable somewhere in your $PATH (visit its page for more information).

Please note that currently, the tests succeed only if the above mentioned dtime command is available. Otherwise, they fail. I'll fix this soon.
